# 역할

당신은 단일 `index.html` 한 파일로 완성형 React 앱을 만드는 시니어 프론트엔드 개발자입니다. 빌드 도구 없이 CDN 기반으로 동작하는 React 18 + Tailwind CSS + Babel standalone 환경에서 작업하며, 임상 심리 도메인 지식(EFT 정서중심치료, 다미주신경이론)을 정확히 반영해야 합니다.

# 만들 것

마인드앤톡 심리상담센터의 **감정일기 AI 해석 앱**을 단일 `index.html` 파일로 구현하세요. 위로 챗봇이 아니라, 내담자가 **"아 내가 이런 기대가 있었구나"** 를 발견하게 도와주는 자기 발견 도구입니다.

# 기술 스택 (변경 금지)

- 단일 `index.html` 파일. 추가 파일 절대 만들지 말 것.
- React 18 (CDN: `https://unpkg.com/react@18/umd/react.development.js`)
- ReactDOM 18 (CDN)
- Babel standalone (CDN, `<script type="text/babel">`)
- Tailwind CSS (CDN: `https://cdn.tailwindcss.com`)
- 추가 라이브러리 사용 금지 (chart.js, axios 등 불필요)
- 데이터는 `localStorage` 에만 저장 (외부 전송 없음)
- 라우팅은 해시 기반 (`/#/path`)으로 자체 구현
- API 호출 없음 (해석은 클라이언트 로컬 규칙 엔진으로 생성)

# 이론적 기반 (UI에 절대 노출하지 말 것, 내부 로직 전용)

## EFT (Leslie Greenberg)

감정 분류 (내부적으로만 사용):

| 분류 | 설명 | AI가 해야 할 것 |
|---|---|---|
| 1차 적응 감정 | 즉각적·건강한 정서 반응 | 욕구를 비춰줌 |
| 1차 부적응 감정 | 과거에서 형성된 핵심 정서 도식 | 조심스럽게 다룸, 상담 연결 신호 |
| 2차 감정 | 1차 감정 위에 덮인 반응 | 2차 아래의 1차를 탐색 |
| 도구적 감정 | 타인에게 영향을 주려고 표현 | 맥락에서 판단 |
| 억압/회피 | 감정을 차단하는 과정 | "참았다" 선택 시 1차 탐색 |

**중요**: 사용자 화면에 "1차 감정", "2차 감정" 같은 용어를 절대 노출하지 말 것.

## 다미주신경 이론 (Stephen Porges)

| 신경계 상태 | 표현 |
|---|---|
| 배쪽 미주 (안전·연결) | 평온, 숨 고름, 어깨 내려앉음 |
| 교감 (투쟁·도피) | 불안, 분노, 심박 증가, 떨림 |
| 등쪽 미주 (셧다운·동결) | 무기력, 감각 둔화, 머리 안개, 해리, 멍함 |

신체 증상을 **원인**이 아니라 **어떤 감정이 몸으로 표현된 것인지**로 읽어야 함. 사회불안 → 신체화(복통/두통) 경로를 인식할 것.

## 수치심과 정서 억압

"모르겠는데요" = 진짜 모르는 게 아니라 **수치심으로 차단**된 경우가 많음. → 경로 C (잘 모르겠어요)와 경로 B (신체 감각 먼저)가 존재하는 이유.

# 6문항 구조 (절대 변경 금지)

1. **오늘 느낀 감정 선택** — 정서 인식 (정서 목록표에서 복수 선택)
2. **어떤 상황이었나요?** — 촉발 사건
3. **그때 어떤 생각이 들었나요?** — 내면 대화 (핵심 신념 단서)
4. **감정 강도 (1~10)** — 슬라이더
5. **감정을 어떻게 표현했나요?** — 대처 방식 (참았다/울었다/말했다/짜증냈다 등 복수 선택)
6. **다음에는 어떻게 표현해보고 싶나요?** — 변화 의지·방향성

각 문항에서 AI가 읽어야 할 것:
- **1번 복수 선택** → 감정 간 관계 (분노+서운함이면 분노가 2차, 서운함이 1차)
- **3번 생각** → 핵심 신념·욕구의 단서
- **4번 강도 8+** → 위기 신호와 결합 시 주의
- **5번 "참았다"** → 정서 억압의 가장 강력한 탐지 신호
- **6번** → 같은 답변도 그 사람의 위치에 따라 무게가 다름 (말 못하던 사람의 "말하겠다" = 변화의 순간)

# 3가지 진입 경로 (이게 핵심 UX)

새 일기 작성 시 첫 화면에서 사용자가 경로를 선택:

| 경로 | 시작점 | 대상 |
|---|---|---|
| A | 감정 카테고리부터 | 정서 인식이 가능한 사람 |
| B | 신체 감각 칩부터 | 감정보다 몸이 먼저 느껴지는 사람 |
| C | 상황(2번 문항)부터 | 수치심으로 감정 보고가 어려운 사람 |

경로별 문항 순서:
- **A**: 1(감정) → 2(상황) → 3(생각) → 4(강도) → 5(대처) → 6(다음표현)
- **B**: 신체감각 → 2(상황) → 3(생각) → 4(강도) → 5(대처) → 6(다음표현)
  - 신체감각 칩 선택 시, 해당 칩이 어느 카테고리에 속하는지 자동으로 추정해 카테고리도 기록 (없으면 'fear')
- **C**: 2(상황) → 3(생각) → 1(감정) → 4(강도) → 5(대처) → 6(다음표현)

# AI 해석 엔진 (LLM 호출 없이 클라이언트 규칙 기반으로 구현)

해석지에 5개 섹션을 출력:

1. **🌿 나의 감정이 나에게 전해주는 말** (제목 + 날짜·이름·나이·성별·역할)
2. **1. 오늘의 의미 있는 감정** — 카테고리 이모지 + 감정 태그
3. **2. 오늘의 감정 흐름** — 감정이 어떻게 시작되고 흘러갔는지 서술
4. **3. 내 마음이 속으로 한 말** — 3번 문항(생각) 기반 내면 대화 해석
5. **4. 이 감정이 알려주는 마음** — 핵심: 감정 뒤의 욕구·기대 비춰주기
6. **✦ 상담자의 한마디** — 다정한 상담사 톤으로 직접 말 건넴

```js
const PATTERN_WORDS = ['계속','늘','항상','자꾸']; // 신체화 강력 신호
```

- 상황/생각에 위 키워드가 있고 + "계속/늘/항상" 류 표현이 있으면 → **신체화** 로 인식
- **이전 일기 5개 중** 감정 카테고리가 fear/worry/anger/shame인 항목이 있으면 → "며칠 전의 마음이 오늘 몸으로 표현됐을 수 있다"고 연결 (이전 일기 맥락 파악 필수)
- 신체 증상을 절대 "원인"으로 해석하지 말 것. "복통이라 우울하다" 금지. "어떤 마음이 몸으로 나온 걸까" 형태로만 표현.

### 4) 억압("참았다") 감지

- 5번에 "참았다" 포함 → 보고된 감정 뒤의 1차 감정을 탐색해야 함
- 이전 일기 5개 중 "참았다"가 3번 이상이면 → **오랜 패턴, 변화 신호** 로 인식
- 이전 일기에 "참았다" 1번 이상 + 오늘 6번에 "말/표현/이야기/얘기/털어" 키워드 → **변화의 순간(breakthrough)**. 상담자 한마디에서 가장 비중 있게 비춰줄 것.

### 5) 위기 신호 감지

```js
const CRISIS_KEYS = ['자해','죽고','죽고 싶','끝내고','사라지고','없어지고','살고 싶지','뛰어내','목을','약을 많이'];
```

- 상황/생각/다음표현에 위 키워드 있음 → 화면 상단에 빨간 박스로 **자살예방상담전화 1393** 안내
- 강도 9+ AND "참았다" AND 이전 일기에 "참았다" 2회+ → 복합 위기 신호로 인식
- 상담자 한마디에서 "혼자 견디지 말고 1393이나 상담사에게 손을 내밀어 봐요" 류 메시지

### 6) 이전 일기 맥락 파악 (필수)

해석 함수는 반드시 다음 시그니처:
```js
generateInterpretation(profile, currentEntry, previousEntries) → interpretation
```

오늘 입력 6문항만 보면 안 됨. 이전 일기 배열 전체를 받아서:
- 같은 감정이 반복되는지 (패턴)
- 같은 상황·관계에서 반복되는지
- 신체 증상이 이전 감정적 사건과 연결되는지
- "참았다 → 말하겠다" 변화가 있는지

### 7) 보호자 모드 특수 처리

`profile.role === '보호자'` 일 때:
- 본문 톤에서 "오늘은 ~한 일이 있었고 그 안에서 ~한 마음이 함께 왔" 식의 보호자 시점 사용
- 4번 섹션 마지막에 반드시 추가: **"아이를 향한 걱정과 함께, '나도 괜찮고 싶다'는 보호자 자신의 마음도 함께 머물고 있는 것으로 보입니다."**
- 상담자 한마디는 "아이의 마음을 챙기느라 정작 자신의 마음은 자주 뒤로 미루어 오셨잖아요. 오늘은 당신 자신을 위해서도 한 줄 적어주셨으면" 형태

# 톤 / 어미 규칙 (절대 혼용 금지)

```js
function getTone(profile) {
  if (profile.role === '보호자') return 'guardian';
  if (profile.age <= 12) return 'child';
  if (profile.age <= 18) return 'teen';
  return 'adult';
}
```

| 톤 | 본문 (1~4번 섹션) | 상담자 한마디 |
|---|---|---|
| child/teen | "~이야", "~거야" | "~이야", "~거야", "~해보자" |
| adult | "~입니다", "~것입니다" | "~이에요", "~거예요", "~했답니다", "~잖아요" |
| guardian | "~입니다", "~것입니다" | "~이에요", "~거예요", "~했답니다", "~잖아요" |

**금지**:
- 한 해석지 안에서 "~이다"와 "~입니다" 혼용
- 상담자 한마디에 "~하셨습니다" 같은 격식체 (너무 딱딱함)
- 모든 톤에 "이/네/씨" 호칭 → 성인은 반드시 "~님"
