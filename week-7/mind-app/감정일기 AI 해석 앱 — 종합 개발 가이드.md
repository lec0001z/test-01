**ⓒ 2026 마인드앤톡 심리상담센터 · 기획: 이지은 상담사**

## 1. 이 앱은 무엇인가

### 한 문장 정의

> **내담자가 감정을 기록하면, AI가 EFT(정서중심치료) 관점에서 "그 감정 뒤에 숨은 욕구와 기대"를 비춰주는 감정일기 앱.**

### GPT 위로 챗봇과의 차이

| 일반 AI 챗봇 | 이 앱 |
|---|---|
| "많이 힘드셨겠네요" (위로) | "아~ 내가 이런 기대가 있었구나" (자기 발견) |
| 오늘 입력만 봄 | 이전 일기 + 오늘 일기를 연결해서 맥락 파악 |
| 감정을 그대로 반영 | 감정 뒤의 욕구·기대·패턴을 해석 |
| 누구에게나 같은 톤 | 나이대·관계·상황에 맞는 맞춤 언어 |

**핵심 원칙: 내담자는 단순한 위로를 넘어 "나를 알고 싶어" 한다. "아~ 내가 이런 기대가 있었구나"를 발견하게 하는 것이 이 앱의 존재 이유다.**

## 2. 이론적 기반

### 2-1. EFT (정서중심치료, Emotion-Focused Therapy)

레슬리 그린버그(Leslie Greenberg)의 EFT에서 감정은 다음과 같이 분류된다:

| 분류 | 설명 | AI가 해야 할 것 |
|---|---|---|
| **1차 적응 감정** | 상황에 대한 즉각적이고 건강한 정서 반응 | 그 감정이 가리키는 욕구를 비춰줌 |
| **1차 부적응 감정** | 과거 경험에서 형성된 핵심 정서 도식. 자동 활성화 | 조심스럽게 다루되, 상담 연결 신호로 판단 |
| **2차 감정** | 1차 감정에 대한 반응. 1차를 덮거나 방어 | 2차 감정 아래의 1차 감정을 탐색 |
| **도구적 감정** | 타인에게 영향을 주기 위해 표현하는 감정 | 해당 시 맥락에서 판단 |
| **정서 억압/회피** | 감정이 올라오려 할 때 차단하는 과정 | "참았다" 선택 시 억압된 1차 감정 탐색 |

**AI는 이 분류를 내부 로직으로만 사용하고, 사용자에게는 절대 노출하지 않는다.**

### 2-2. 다미주신경이론 (Polyvagal Theory)

스티븐 포지스(Stephen Porges)의 이론. 자율신경계의 세 가지 상태:

| 신경계 상태 | 작동 | 신체 표현 | 정서 표현 |
|---|---|---|---|
| **배쪽 미주 (Ventral Vagal)** | 안전·연결 | 숨 고르게, 긴장 풀림, 어깨 내려앉음 | 평온, 기쁨, 소속감 |
| **교감신경 (Sympathetic)** | 투쟁·도피 | 심박 증가, 근육 긴장, 호흡 얕아짐, 떨림 | 불안, 분노, 두려움 |
| **등쪽 미주 (Dorsal Vagal)** | 셧다운·동결 | 몸에 힘 빠짐, 감각 둔화, 머리 안개 낀 느낌 | 멍함, 무력감, 해리 |

**AI 해석에서의 활용:**
- 신체 증상이 보고될 때, 어떤 신경계 상태인지 내부적으로 판단
- 특히 **사회불안 → 신체화(복통, 두통 등)** 경로를 인식해야 함
- 신체 증상을 "원인"으로 보지 말고, "어떤 감정이 몸으로 나온 것인지" 탐색

### 2-3. 수치심과 정서 억압

임상에서 가장 흔한 패턴:

> 내담자가 "모르겠는데요"라고 할 때, 대부분은 진짜 모르는 게 아니다.
> 느끼고 있지만 말하면 부끄러우니까 숨기는 것이다.
> **수치심이 감정 보고 자체를 막는 구조.**

| 표면 | 실제 |
|---|---|
| "모르겠는데요" | 느끼고 있지만 드러내면 부끄러움 |
| "그냥요" | 감정이 있지만 약해 보일까봐 차단 |
| "괜찮아요" | 괜찮지 않지만 감정을 보이는 것 자체가 수치스러움 |

**앱 설계 시사점:** "잘 모르겠어요" 경로(경로 C)가 존재하는 이유. 신체 감각 먼저 시작하는 경로 B도 수치심 우회로.

## 3. 감정일기 6문항 구조

| 문항 | 질문 | 목적 |
|---|---|---|
| 1번 | 오늘 느낀 감정 선택 | 정서 인식 (정서 목록표에서 복수 선택) |
| 2번 | 어떤 상황이었나요? | 촉발 사건 파악 |
| 3번 | 그때 어떤 생각이 들었나요? | 내면 대화 (핵심 신념 단서) |
| 4번 | 감정 강도 (1~10) | 강도 측정 |
| 5번 | 감정을 어떻게 표현했나요? | 대처 방식 (참았다/울었다/말했다/짜증냈다 등) |
| 6번 | 다음에는 어떻게 표현해보고 싶은가요? | 변화 의지·방향성 |

### 각 문항에서 AI가 읽어야 할 것

**1번:** 선택된 감정이 2차 감정인지 판단. 복수 선택 시 감정 간 관계(예: 분노+서운함 → 분노가 2차, 서운함이 1차).

**2번:** 상황의 관계적 맥락. 누가 관련되어 있는지, 반복되는 상황인지.

**3번:** 핵심 신념의 단서. "내가 잘못했나?", "태어난 죄밖에 없다" 같은 표현에서 핵심 욕구를 읽어냄.

**4번:** 강도 8 이상은 높은 정서적 각성. 위기 신호와 결합 시 주의.

**5번:** **"참았다"는 정서 억압의 가장 강력한 탐지 신호.** AI가 이걸 보면 "보고된 감정 뒤에 억압된 1차 감정이 있을 수 있다"고 탐색해야 함.

**6번:** 단순한 계획이 아니라, **그 사람의 현재 위치에서 읽어야 함.** 말을 못 하던 사람이 "말하겠다"고 하면 변화의 순간. 원래 말을 잘 하는 사람이 "말하겠다"고 하면 기존 패턴 유지.

## 4. 정서 목록표 (9개 카테고리)

### 제시 방식

- 9개 카테고리를 카드 형태로 먼저 보여줌
- 카테고리를 탭하면 **심리 언어 + 신체 감각을 구분 없이 한 흐름으로 나열** (칩/태그)
- 몸과 마음을 나누지 않음 — EFT와 다미주신경이론의 핵심이 "몸과 마음은 분리되지 않는다"이므로
- 복수 선택 가능

### 3가지 진입 경로

| 경로 | 진입 | 대상 |
|---|---|---|
| A | 감정 카테고리 먼저 | 정서 인식이 가능한 사람 |
| B | 신체 감각 먼저 | 감정보다 몸의 느낌이 먼저 오는 사람 |
| C | "잘 모르겠어요" | 수치심으로 감정 보고가 어려운 사람 → 상황(2번)부터 시작 |

### 9개 카테고리 목록

| # | 카테고리 | 한 줄 설명 |
|---|---|---|
| ① | 기쁨 / 활력 | 기분 좋고 에너지가 느껴질 때 |
| ② | 슬픔 / 상실감 | 마음이 가라앉고 허전할 때 |
| ③ | 불안 / 두려움 | 지금 무섭거나 긴장될 때 (즉각적) |
| ④ | 걱정 / 염려 | 앞으로의 일이 신경 쓰일 때 (미래 지향적) |
| ⑤ | 분노 / 좌절 | 화나거나 답답할 때 |
| ⑥ | 수치 / 부끄러움 | 창피하거나 숨고 싶을 때 |
| ⑦ | 당황 / 혼란 | 어쩔 줄 모르겠을 때 |
| ⑧ | 애착 / 갈망 | 누군가 곁에 있었으면 할 때 |
| ⑨ | 평온 / 안정감 | 마음이 편안하고 고요할 때 |

**③과 ④를 분리한 이유:** "지금 무섭다"(즉각적 두려움)와 "앞으로 어떡하지"(미래 지향적 염려)는 다른 감정이고, 다른 욕구를 가리킴.

**⑧이 존재하는 이유:** EFT에서 애착 갈망은 감정이다. "인정받고 싶음", "이해받고 싶음"은 욕구가 아니라 정서적 경험. 내담자가 직접 자기 욕구를 감정으로 인식하게 하는 장치.

### 다미주신경이론 관점 보완

②번 카테고리에 등쪽 미주(셧다운) 상태의 신체 감각 포함:
- 몸에 힘이 빠지는 느낌
- 감각이 둔해지는 느낌
- 머리가 안개 낀 느낌

## 5. AI 해석지 구조

### 해석지 섹션 구성

| 섹션 | 내용 |
|---|---|
| 제목 | 🌿 나의 감정이 나에게 전해주는 말 |
| 날짜·이름 | 2025. 10. 6. · 김혜영님 (32세, 여) |
| 1. 오늘의 의미 있는 감정 | 이모지 + 감정 태그 |
| 2. 오늘의 감정 흐름 | 감정이 어떻게 시작되고 흘러갔는지 서술 |
| 3. 내 마음이 속으로 한 말 | 3번 문항 기반 내면 대화 해석 |
| 4. 이 감정이 알려주는 마음 | **핵심: 감정 뒤의 욕구·기대 비춰주기** |
| ✦ 상담자의 한마디 | 다정한 상담사 톤으로 직접 말 건넴 |

### 나이대별 문체 기준

| 나이대 | 해석 본문 (1~4번) | 상담자 한마디 |
|---|---|---|
| 초등 | "~이야", "~거야" | "~이야", "~거야", "~해보자" |
| 청소년 | "~이야", "~거야" | "~이야", "~거야", "~해보자" |
| 성인/대학생 | "~입니다", "~것입니다" | "~이에요", "~거예요", "~했답니다", "~잖아요" |
| 부모/보호자 | "~입니다", "~것입니다" | "~이에요", "~거예요", "~했답니다", "~잖아요" |

**문체 통일 원칙:** 한 해석지 안에서 "~이다"와 "~입니다"가 섞이면 안 됨. 상담자 한마디는 "~하셨습니다"(너무 격식)가 아니라 "~했답니다"(다정한 상담사 말투).

## 6. AI 해석의 핵심 원칙

### 6-1. 맥락 파악 (가장 중요)

**오늘 입력된 6문항만 보면 안 된다. 이전 일기 전체를 맥락으로 넘겨야 한다.**

| 원칙 | 설명 | 예시 |
|---|---|---|
| 이전 일기 연결 | 같은 감정이 반복되면 패턴으로 읽음 | 9/11 사회불안 → 9/15 복통 = 같은 불안의 두 표현 |
| 사람 단위로 읽기 | "이 사람이 어떤 사람이었는지" 위에서 오늘을 읽음 | 말 못하던 사람의 "말하겠다" = 변화의 순간 |
| 6번의 무게 | 같은 답변도 누가 말하느냐에 따라 무게가 다름 | 원래 말 잘하는 사람 vs 참아온 사람 |

### 6-2. 신체화 인식

**신체 증상이 먼저 보고될 때, 그것을 원인으로 보지 말고 — 이전 일기와 맥락을 연결해서 어떤 감정이 몸으로 나온 것인지 먼저 탐색할 것.**

| 잘못된 해석 | 올바른 해석 |
|---|---|
| "배가 아파서 우울한 거구나" | "어떤 마음이 몸으로 나온 건지" 탐색 |
| 복통 = 신체 증상 | 복통 = 사회불안의 신체 표현 가능성 |
| "계속"을 무시 | "계속"은 신체화의 강력한 신호 |

### 6-3. 2차 감정 뒤의 1차 감정 탐색

| 보고된 감정 (2차) | 탐색할 1차 감정 |
|---|---|
| 분노 | 서운함, 상처, 무시당한 느낌 |
| 짜증 | 두려움, 걱정 (특히 보호자의 경우) |
| 무감각/멍함 | 억압된 슬픔 또는 두려움 |
| 자책 | 사랑받고 싶은 마음 (관계 유지 욕구) |

### 6-4. "참았다"의 의미

5번에서 "참았다"를 선택한 경우:
- 정서 억압의 직접적 신호
- "보고된 감정 뒤에 억압된 1차 감정이 있을 수 있다"
- "늘, 항상 참는다"는 표현 → 오랜 패턴, 변화 필요 신호

### 6-5. 보호자 케이스 특수 원칙

부모/보호자가 자녀에 대해 쓸 때:

| 원칙 | 설명 |
|---|---|
| 보호자의 감정도 독립적으로 다룸 | "아이 걱정"만이 아니라 "나도 괜찮고 싶다"는 마음 |
| 짜증/분노의 재해석 | 자녀를 향한 짜증 = 대부분 두려움의 표현 |
| "참았다"의 무게 | 보호자의 "참았다"는 오랜 돌봄 피로의 신호 |
| 변화 인식 | "예전과 다르게 대화가 됐다" = 보호자의 노력 결과도 비춰줌 |

## 7. 위기 신호 감지

### 감지 기준

| 신호 | 대응 |
|---|---|
| 자해/자살 관련 표현 | 위기 상담 기관 안내 (자살예방상담전화 1393) |
| 감정 강도 9~10 + "참았다" + 반복 패턴 | 상담 연결 권유 |
| 등쪽 미주 셧다운 징후 (무감각, 해리) | 부드럽게 전문 상담 안내 |

### 주의사항

- "태어난 죄밖에 없다" 같은 표현 → 맥락에서 판단. 관계적 고통의 표현인지, 자기 존재를 끝내고 싶다는 신호인지.
- 보호자가 자녀의 자해를 보고하는 경우 → 보호자의 감정으로 읽되, 자녀의 안전에 대한 안내도 포함.

## 8. 감정 패턴 분석

일기가 쌓이면 AI가 패턴을 분석:

| 분석 항목 | 내용 |
|---|---|
| 반복되는 감정 | 최근 7일간 가장 많이 선택된 감정 |
| 반복되는 상황 | 같은 관계/맥락에서 같은 감정이 반복되는지 |
| 대처 방식 변화 | "참았다"에서 "말했다"로 변화 → 긍정적 변화로 비춰줌 |
| 신체화 패턴 | 특정 감정 후 반복적으로 신체 증상이 나타나는지 |

## 9. 테스트 케이스 (실제 논의된 사례)

### 케이스 1: 서예준 (중3, 남)

**9/11 일기:**
- 감정: 무섭다
- 상황: 친구가 나쁜 일을 했는데 나도 같이 한 것으로 오해받을까봐
- AI가 읽어야 할 것: "무섭다"의 실체는 **사회불안** (타인의 시선 + 오해에 대한 두려움). 핵심 욕구: "결백하게 지켜지고 싶다"

**9/15 일기:**
- 감정: 우울
- 상황: 계속 배가 아파서
- AI가 읽어야 할 것: **9/11 사회불안 → 9/15 복통 = 신체화.** "계속"이 핵심. 복통을 원인으로 보지 말고, 불안이 몸으로 나온 것으로 읽어야 함.

### 케이스 2: 송미선 (학부모, 아들 서예준)

- 감정: 걱정, 두려움, 불안 (강도 10)
- 상황: 아침에 눈 뜨자마자 아이 걱정
- 대처: "늘, 항상 나는 참는다"
- AI가 읽어야 할 것: "아이가 괜찮기를 바란다" + **"나도 괜찮고 싶다"** (보호자 자신의 욕구). "참는다"의 반복 → 돌봄 피로.

### 케이스 3: 김혜영 (32세, 여)

- 감정: 외로움, 무력감, 서운함, 못난 느낌, 자책 (강도 10)
- 상황: 아빠가 엄마 이야기 후 차갑게 대함
- 생각: "태어난 죄밖에 없는데, 내가 잘못했나?"
- 6번: "나는 다음에도 여전히 내 속이야기를 할 것이다"
- AI가 읽어야 할 것: 자책 = 아빠의 사랑을 잃고 싶지 않아서. 6번 = **말 못하던 사람의 변화의 순간** (가장 크게 비춰야 함).

### 케이스 4: 정연옥 (보호자, 딸 정지민 고1)

- 감정: 우려, 어이없다, 긴장, 다행스럽다 (강도 7)
- 상황: 응급실 다녀온 딸이 미용시술 이야기, ADHD 약 처방 후 귀가
- 대처: 짜증냈다 + 말로 이야기했다
- AI가 읽어야 할 것: 짜증 = 딸이 다시 불안정해질까 봐 나온 **두려움**. "다행스럽다"가 마지막에 있음 = 변화의 신호. 보호자의 노력도 비춰줘야 함.

## 10. 절대 하면 안 되는 것

| 금지 사항 | 이유 |
|---|---|
| "많이 힘드셨겠네요" 식 단순 위로 | GPT가 하는 것. 이 앱의 존재 이유가 아님 |
| EFT 용어 노출 ("1차 감정", "2차 감정") | 내담자에게 이론 용어는 불필요 |
| 감정을 판단하거나 교정 | "그렇게 느끼면 안 돼요" 절대 금지 |
| 조언/해결책 제시 | 상담사가 아님. 비춰주기만 함 |
| 이전 일기 무시 | 맥락 없이 오늘만 보면 해석 품질 급락 |
| 신체 증상을 원인으로 단정 | "배 아파서 우울한 거구나" 금지 |
| "~하셨습니다" 격식체 | 다정한 상담사 톤 유지 |
| 문체 혼용 | 한 해석지 안에서 "~이다"와 "~입니다" 섞지 않음 |

## 11. 기술 스택 및 구현 참고

- React Native + Expo SDK 54

### AI 해석 호출 시 전달해야 할 데이터

```
{
  profile: { name, age, gender, role },
  currentEntry: { emotions, situation, thoughts, intensity, coping, nextExpression },
  previousEntries: [ 최근 5~10개 일기 전체 ],
  patterns: { frequentEmotions, frequentCoping, bodySymptoms }
}
```

**이전 일기를 반드시 함께 전달해야 맥락 파악이 가능하다.**
