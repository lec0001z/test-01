## 1. 기술 스택 (확정)

| 영역 | 선택 | 근거 |
|---|---|---|
| 프레임워크 | **Next.js 16 (App Router)** | week-5 record-shop에서 검증됨, Server Actions로 빠른 풀스택 |
| DB / Auth | **Supabase** (Postgres + Auth + RLS) | week-5에서 사용 경험 ↑, 무료 티어로 MVP 충분 |
| LLM | **Anthropic Claude API** (`claude-haiku-4-5`) | 한국어 품질 우수, Haiku로 비용·지연 최소화. 분석 정밀도 필요해지면 Sonnet 4.6 |
| 배포 | **Vercel** | Next.js 네이티브, 프리뷰 URL로 데모 데이 시연 |
| 스타일 | Tailwind + shadcn/ui | 빠른 컴포넌트, 한국어 폰트 적용 쉬움 |

**Out of scope (MVP 이후)**: 모바일 네이티브, PWA, 결제, 음성 입력, 치료사 공유.

## 2. MVP 범위 — 3주 안에 꼭 넣을 핵심 기능 3개

> 이 3개 외의 모든 기능은 **MVP 이후로 미룬다**. 의심되면 자른다.

### ✅ 핵심 기능 ① — **자연어 일기 입력 → AI 분석 카드**
- 입력: "오늘 어떤 상황 + 어떤 감정" 자유 텍스트 1개 폼
- 출력: LLM이 분석해 카드 3개 리턴
  - **인지 왜곡 식별** (흑백논리/과잉일반화/자기비난 등 1~2개)
  - **감정-상황 트리거 요약** (1줄)
  - **리프레임 질문** 1~2개
- 응답 시간 목표: **5초 이내** (스트리밍 표시)
- *왜 핵심?* — 차별화 ① (research.md §3). 이게 없으면 그냥 메모장.

### ✅ 핵심 기능 ② — **일기 목록 + 자동 저장 + 한국어 네이티브 UX**
- 목록 페이지: 카드형, 날짜·감정·요약 한 줄·인지왜곡 태그
- 상세 페이지: 원문 + AI 분석 카드 다시 보기
- **타이핑 중 2초마다 로컬 자동 저장** (Reflectly의 *"30분 쓴 일기 사라짐"* 리뷰 학습)
- 모든 마이크로카피·예시 질문 한국어 네이티브 (직장/가족/학업 컨텍스트)
- *왜 핵심?* — 차별화 ② + research.md §4-1 데이터 손실 페인 직접 대응.

### ✅ 핵심 기능 ③ — **월간 패턴 리포트 (1화면)**
- "이번 달 자주 등장한 트리거 키워드 Top 5"
- "이번 달 가장 많이 나온 인지 왜곡 패턴" + 자연어 해설 1문단
- 단순 막대그래프 + LLM 자연어 요약 (Daylio처럼 차트만 두지 않는다)
- *왜 핵심?* — 차별화 ③ (research.md §3). "기록 → 통찰" 루프 닫는 마지막 조각.

### MVP 명시적 제외 (Demo 후로)
- 회원 간 공유, 댓글, 챌린지
- 다국어, 다크 테마 커스터마이징
- 이모지/스티커 시스템 (Daylio 영역 — 안 싸움)
- 결제·구독 (무료로 데모, 가격은 mission.md §6 이후)
- 음성 입력, 사진 첨부

## 3. 데이터 모델 (초안)

```
users (Supabase Auth)
  ├─ id
  └─ email

entries
  ├─ id (uuid)
  ├─ user_id (fk)
  ├─ created_at
  ├─ raw_text          -- 사용자 원문
  ├─ situation         -- LLM 추출 A (Activating event)
  ├─ belief            -- LLM 추출 B (Belief/생각)
  ├─ consequence       -- LLM 추출 C (감정/행동)
  ├─ distortions       -- text[] (예: ["흑백논리", "자기비난"])
  ├─ reframe_questions -- text[]
  └─ summary           -- 1줄 요약

monthly_reports (캐시)
  ├─ user_id, year_month (pk)
  ├─ top_triggers      -- jsonb
  ├─ top_distortions   -- jsonb
  └─ narrative         -- LLM 자연어 요약 1문단
```

RLS: 모든 테이블 `user_id = auth.uid()` 정책으로 잠금.
