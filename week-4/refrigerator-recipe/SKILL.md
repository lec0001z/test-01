# 냉장고 재료 기반 레시피 생성기

사용자의 `week-4/refrigerator-recipe/ingredients/` 폴더에 있는 재료 JSON 파일들을 읽어 레시피를 제안하고, 선택된 레시피를 `week-4/refrigerator-recipe/recipes/` 폴더에 마크다운으로 저장합니다.

## 경로 규칙

- **재료 폴더**: `week-4/refrigerator-recipe/ingredients/`
- **레시피 저장 폴더**: `week-4/refrigerator-recipe/recipes/`
- 아래 모든 상대 경로는 위 기준으로 해석합니다.

## 재료 JSON 스키마

`week-4/refrigerator-recipe/ingredients/<name_en>.json` 한 재료당 1파일:

```json
{
  "name": "한글 이름",
  "name_en": "english_name",
  "category": "카테고리 (단백질/채소/곡물/유제품/육류/발효식품 등)",
  "quantity": 숫자,
  "unit": "개|g|ml|공기|모|대|통|장",
  "expiration": "YYYY-MM-DD",
  "storage": "냉장|냉동|실온",
  "notes": "자유 텍스트 메모"
}
```

## 실행 순서

`/recipe` 가 호출되면 **반드시 아래 순서대로** 수행하세요.

### 1. 재료 스캔

- `Glob` 로 `week-4/refrigerator-recipe/ingredients/*.json` 전체 경로를 수집.
- 각 파일을 `Read` 로 읽고 JSON 파싱. 실패한 파일은 무시하고 경고만 출력.
- 결과를 하나의 "재료 테이블"로 정리 (이름, 수량+단위, 유통기한, 카테고리).

### 2. 우선순위 판단

아래 기준으로 **이번에 쓸 재료 셋**을 선택:

1. **유통기한 임박 재료 우선** — 오늘 날짜 기준으로 남은 일수가 짧은 재료(특히 3일 이내)를 필수 재료로 끌어올림.
2. **카테고리 밸런스** — 단백질 1개 + 채소 1~2개 + (선택) 곡물/유제품 1개 조합이 되도록.
3. **사용자 요청 반영** — 사용자가 "한식/양식", "매운 거", "15분 안", "비건" 같은 조건을 줬다면 최우선 적용.

### 3. 레시피 제안 (자동 선택)

- 후보 2~3개를 **속으로 고려만** 하고, **가장 추천하는 1개**만 발표.
- 발표 형식: 메뉴 이름 + 한 줄 추천 이유(임박재료 소진 등) + 사용 재료 리스트 + 예상 조리시간.
- **절대 사용자에게 "어떤 걸로 할까?" 묻지 않는다.** 바로 4단계 저장으로 진행.
- 사용자가 메뉴·스타일·제약을 직접 지정한 경우(예: `/recipe 매운거`, `/recipe 김치 꼭 써서`)는 그 조건에 가장 부합하는 1안을 자동 선택.
- 사용자가 나중에 "다른 거 줘", "2안으로 해줘" 같이 명시적으로 교체 요청하면 그때 다음 후보로 재작성.

### 4. 레시피 마크다운 저장

선택된 레시피를 다음 형식으로 `week-4/refrigerator-recipe/recipes/` 폴더에 저장:

- **파일명 규칙**: `week-4/refrigerator-recipe/recipes/YYYY-MM-DD_<slug>.md` (예: `week-4/refrigerator-recipe/recipes/2026-04-24_kimchi-fried-rice.md`)
  - `<slug>`는 영문 소문자 + 하이픈. 한글 메뉴명도 영문 슬러그로 변환 (예: 김치볶음밥 → `kimchi-fried-rice`).
  - 같은 날 같은 슬러그가 이미 존재하면 뒤에 `-2`, `-3` 붙임.
- **필수 마크다운 구조**:

```markdown
# <요리 이름>

> <한 줄 설명>

- **조리시간**: <예상 분>
- **난이도**: <쉬움/보통/어려움>
- **인분**: <몇 인분>
- **생성일**: <YYYY-MM-DD>

## 사용한 냉장고 재료

| 재료 | 사용량 | 보유량 | 유통기한 |
|------|--------|--------|----------|
| ... | ... | ... | ... |

## 추가로 필요한 재료

- (없으면 "냉장고 재료만으로 충분" 이라고 적기)

## 조리 순서

1. ...
2. ...
3. ...

## 팁

- ...
```

- `Write` 툴로 해당 경로에 마크다운 파일을 생성.
- `recipes/` 폴더가 없으면 생성 후 쓰기.

### 5. 사용자에게 결과 보고

최종적으로 짧게 요약:

- 저장된 파일 경로 (`recipes/…md`)
- 이번에 사용한 재료 개수와 유통기한 임박 재료 몇 개를 "구해냈는지"
- 남은 재료로 다음에 추천 가능한 다른 레시피 1개를 한 줄로 힌트.

## 규칙과 주의사항

- **실제 없는 재료를 레시피에 넣지 마세요.** 냉장고에 있는 재료만 "사용한 냉장고 재료" 표에 넣고, 부족한 것은 "추가로 필요한 재료" 섹션에 분리.
- 사용량은 보유량을 넘지 않게 계산.
- 유통기한이 이미 **지난** 재료는 레시피에서 제외하고 사용자에게 "폐기 권장" 으로 별도 안내.
- 레시피는 실제 조리 가능한 순서로 작성 (전처리 → 가열 → 양념 → 마무리).
- 조리 단계는 한국어로, 명령형 문장으로 간결하게.
- 사용자가 특정 재료를 지정하면 그 재료를 무조건 포함시킵니다.
