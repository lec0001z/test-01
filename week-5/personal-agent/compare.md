# Before / After — 같은 질문에 답변이 어떻게 달라지는가

> 이 문서는 미션 4(Before/After 비교 시연)의 결과입니다.
> 동일한 질문 3개를 **(A) Context 없이** vs **(B) Context + DB 있이** 두 환경에서 던져 답변을 비교합니다.

---

## 시연 환경

| 환경 | 설명 |
|---|---|
| (A) Context 없음 | 빈 폴더 / 일반 채팅. AI는 사용자에 대해 아무것도 모름. |
| (B) Context + DB 있음 | `week-5/personal-agent/` 안에서 작업. `CLAUDE.md` 가 자동으로 `context.md` + `data/*.json` 4개를 읽음. |

## 질문 1 — "오늘 뭐 하면 좋을까?"

### (A) Context 없이

> 오늘 뭐 할지 정하실 때는 보통 ① 해야 할 일 ② 하고 싶은 일 ③ 컨디션 세 가지를 균형 있게 보면 좋습니다.
> 일정에 마감이 있는 일이 있다면 그것 먼저, 없다면 산책이나 독서처럼 기분 환기에 좋은 활동도 좋아요.
> 좀 더 구체적으로 추천 받고 싶으면 오늘 일정과 컨디션을 알려주세요.

→ **일반론**. 누구한테 해도 같은 답.

### (B) Context + DB 있이

> 오늘 토요일이니까 **6주차 수업 듣기** 가 1순위 (`context.md` "단기 목표"). 끝나면 곧장 **수학 학원 월말 모의고사** 가 P1 — 점수 떨어지면 반 강등이라 했던 그 시험.
> 시험 끝나고 시간 남으면, 어제 한로로 워처가 알려준 `[DEEP TALK] 애증` (`hanroro-listen-log` 5/1자) 을 작곡 참고용으로 한 번 더 보길. 마침 `practice-log` 에 "B파트 코드 진행이 안 잡힘" 블로커가 있어서, 한로로가 직접 곡 만든 과정을 듣는 게 도움 될 가능성 높음.
> 친구 생일 선물도 오늘 사야 함 (내일 파티).
>
> *(based on: context.md 일정, hanroro-listen-log 2026-05-01, practice-log.summary.blocker)*

→ **개인화**. 본인 진짜 일정 + 강의 진도 + 작곡 블로커까지 연결.

## 질문 2 — "한로로 노래 한 곡만 추천해줘"

### (A) Context 없이

> 한로로 추천곡이라면 입문곡으로는 `0+0` 이나 `시간을 달리네` 가 무난합니다.
> 가사가 깊은 걸 좋아하면 `ㅈㅣㅂ` 도 추천드립니다.
> 어떤 분위기의 곡을 원하시나요?

→ 일반적인 한로로 추천 리스트.

### (B) Context + DB 있이

> 이번 주 새로 들은 곡 중에 가장 별 5개 준 게 **"게임 오버 ?"** (4/30, `hanroro-listen-log`) — "MV 4주 만에 516만, 가사 마음에 듦" 이라고 본인이 코멘트.
> 다시 한 번 들어볼 시점은 **오늘 학원 가는 길 또는 점심**. `0+0` 은 이미 즐겨찾기(favorite_track)이니 새로움이 없음.
> 작곡 연습이 5/3 일요일 잡혀 있으니, 그 전에 한 번 더 듣고 인상에 남은 코드 흐름을 메모해두면 `practice-log` B파트 작업에 바로 쓸 수 있음.
>
> *(based on: hanroro-listen-log.summary.most_recent_discovery, practice-log 일정)*

→ 본인이 이미 들은 곡 / 안 들은 곡을 구분하고, 작곡 연습과 연결까지.

## 질문 3 — "내가 지금까지 뭐 했는지 한 줄로 정리해줘"

### (A) Context 없이

> 사용자님이 어떤 활동을 해오셨는지 제가 직접 알기는 어렵습니다.
> 최근 한 일들을 알려주시면 정리해드릴 수 있어요.

→ 답변 불가.

### (B) Context + DB 있이

> 이은찬님은 **2026-04-04 부터 현재까지 5주 동안 강의를 들으며 자동화/AI 도구 학습** 을 이어오고 있어요. (`study-log`: 총 780분, 4주차 완료 + 5주차 진행 중) 가장 좋아한 주제는 **MCP / 자동화** 였고, 약한 영역은 **DB 스키마 설계**.
> 음악 쪽으로는 **2026-03-29 부터 매주 일요일 GarageBand 작곡 연습** 5회 진행 (`practice-log`: 285분, 가제 "무제 1번", B파트 막힘).
> 한로로 청취는 4월 한 달간 7회, 평균 별점 4.7, 메인 발견은 **"게임 오버 ?"**.
> 가장 최근에는 한로로 자동 워처와 본 personal-agent를 직접 만듦.
>
> *(based on: study-log + practice-log + hanroro-listen-log + context.md 진행 중인 프로젝트)*

→ 4개 파일을 모두 합쳐 본인의 활동 히스토리를 한 번에 그림.

---

## 결론

| 측면 | Context 없이 | Context + DB 있이 |
|---|---|---|
| 답변 시간 | 즉답 가능 | 거의 동일 (파일 4개 읽는 데 1초 미만) |
| 정확도 | 일반론에 그침 | 본인 실제 일정·기록·취향 인용 |
| 활용도 | 다시 질문해야 함 | 바로 행동으로 연결됨 |
| 신뢰감 | "AI는 나를 모름" | "AI가 내 컨텍스트를 안다" |

**핵심:** Context.md 1장 + 작은 JSON DB 3개만으로 일반 챗봇 → 맞춤형 비서로 격이 올라감.
